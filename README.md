## Trabajo de Investigación

### Indicaciones

- A realizarse en grupos de 3.
- Se deberá elegir un tema entre los listados y realizar un trabajo de investigación sobre los conceptos que se les solicitan.
- Los entregables deben ser:
    - Documento formal de los resultados de investigación.
    - Documento con estándares, recomendaciones o reglas para ser aplicado en el proyecto.
    - Exposición de no más de 20 minutos donde expongan los hallazgos.

#### Estándares de código Java (Grupo _)

- Convenciones de código Java.
    - Describir las reglas más importantes.
    - Recomendaciones de calidad de código.
- Herramientas para análisis estático de código.
    - Automatización de la verificación de estándares de código en proyectos Java.

#### Estándares de código Typescript (Grupo _)

- Reglas y convenciones comunes en TypeScript.
- Herramientas de linting y análisis estático para TypeScript.
- Integración de TypeScript en proyectos web.
- Comparación de TypeScript con otros lenguajes de programación.

#### Estándar de Branching strategy (Grupo _)

- Tipos de estrategias de ramificación (GitFlow, GitHub Flow, etc.).
- Ventajas y desventajas de diferentes estrategias de ramificación.
- Resolución de conflictos y manejo de ramas en equipos colaborativos.
- Herramientas y prácticas recomendadas para gestionar ramas de código.

#### Estándares de Interfaz de usuario (Grupo _)

- Principios de diseño de interfaz de usuario (UI) y experiencia de usuario (UX).
- Herramientas y tecnologías para el diseño de UI/UX.
- Diseño adaptable y responsivo.
- Accesibilidad en el diseño de interfaces de usuario.

#### Estándares de diseño de base de datos (Grupo _)

- Convenciones de código.
- Optimización de consultas y rendimiento de bases de datos.
- Seguridad y protección de datos en bases de datos.
- Herramientas de libre uso para base de datos relacionales.

#### Recomendaciones para realizar Code Reviews (Grupo _)

- Importancia de las revisiones de código en el desarrollo de software.
- Procesos y mejores prácticas para llevar a cabo revisiones de código efectivas.
- Herramientas y plataformas para facilitar las revisiones de código.
- Resolución de conflictos y comunicación en revisiones de código.

#### Guía de Despliegue (Grupo _)

- Estrategias de implementación de software (despliegue continuo, despliegue gradual, etc.).
- Herramientas y plataformas de automatización de despliegue (CI/CD).
- Consideraciones de seguridad en el despliegue de aplicaciones.
- Monitorización y registro de aplicaciones en producción.
- Estrategias de reversión y manejo de fallos en el despliegue.

#### Estándares de Pruebas y Cobertura de Código (Grupo _)

- Tipos de pruebas (unitarias, integración, ui).
- Herramientas y marcos de trabajo de pruebas (JUnit, Selenium, Cypress, etc.).
- Medición y mejora de la cobertura de código.

### Entregables

- Documento formal de los resultados de investigación(33%).
    - Debe contener una introducción, desarrollo y conclusiones.
    - Debe contener referencias bibliográficas.
    - Debe contener ejemplos prácticos.
- Documento con estándares, recomendaciones o reglas para ser aplicado en el proyecto(33%).
    - Debe ser un documento técnico.
    - Debe contener instrucciónes claras y concisas.
- Exposición de no más de 20 minutos donde expongan los hallazgos(34%).
    - Debe ser una presentación formal.
    - Debe contener ejemplos prácticos.
    - Debe contener una demostración práctica.
    - Debe contener una sección de preguntas y respuestas.

Todos los documentos se deberán entregar en un Pull Request donde se deben crear un subdirectorio en el directorio de investigación con el nombre del proyecto.
